# 104. Maximum Depth of Binary Tree

- **难度**: Easy
- **链接**: https://leetcode.com/problems/maximum-depth-of-binary-tree/

## 思维误区复盘

核心卡点：第一版用「全局变量 + 回溯计数」硬凑深度，既要维护 `ans` 又要手动加减 `cnt`，还踩了闭包赋值、空节点未判等坑。其实树的深度天然适合**自底向上的分治递归**——让 `dfs` 直接返回子树深度，根本不需要任何外部变量。

**❌ 错误写法**

```python
def maxDepth(self, root):
    ans = 0
    cnt = 0
    def dfs(node):
        if node.left is None and node.right is None:  # root 为 None 时直接崩
            ans = max(ans, cnt)        # 闭包内赋值 -> UnboundLocalError
        dfs(node.left)
        cnt += 1                       # 同样 UnboundLocalError；且加减逻辑不完整
        dfs(node.right)
        return ans
    dfs(root)                          # 还漏了 return
```

**✅ 正确写法**

```python
def dfs(node):
    if node is None:                   # base case：空节点深度为 0
        return 0
    return 1 + max(dfs(node.left), dfs(node.right))  # 子树返回值组合
```

## 最终解法

```python
class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        def dfs(node: Optional[TreeNode]) -> int:
            if node is None:
                return 0
            return 1 + max(dfs(node.left), dfs(node.right))
        return dfs(root)
```

- **时间复杂度**: O(n)，每个节点访问一次
- **空间复杂度**: O(h)，递归栈深度等于树高；最坏（链状）O(n)，平衡树 O(log n)
