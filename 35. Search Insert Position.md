# 35. Search Insert Position

> 给定一个**有序数组**和一个目标值，返回目标值的索引。如果不存在，返回它**按顺序插入**的位置。
> 要求时间复杂度 `O(log n)` —— 即二分查找。

链接：https://leetcode.com/problems/search-insert-position/

---

## 一、我最初的错误代码

```python
class Solution:
    def searchInsert(self, nums: List[int], target: int) -> int:
        n=len(nums)
        left=0
        right=n-1
        mid=(right-left)/2
while left<right:
if nums[mid]==target:
return mid
else if nums[mid]<target:
                left=mid
else if nums[mid]>target:
                right=mid
```

### 问题清单

**语法问题**

1. `else if` 在 Python 里不存在，应写 `elif`。
2. 缩进全乱：`while` / `if` / `return` 都顶格了，会报 `IndentationError`。
3. `mid=(right-left)/2`：`/` 得到浮点数（如 `2.0`），不能做索引；要用 `//` 整除。且这个式子算的是「区间长度的一半」，不是中点。

**逻辑问题**

4. `mid` 只在循环外算了一次，循环里 `left`/`right` 变了但 `mid` 没更新 → 死循环或结果错误。`mid` 必须放在循环**内部**每轮重算。
5. `nums[mid] < target` 时写 `left = mid`，若 `mid == left` 则区间永不缩小 → 死循环。应 `left = mid + 1`。
6. 没处理「插入位置」：找不到 target 时要返回插入下标。循环结束后应 `return left`。

## 二、正确代码（左闭右闭模板）

```python
class Solution:
    def searchInsert(self, nums: List[int], target: int) -> int:
        left = 0
        right = len(nums) - 1
        while left <= right:                 # 注意是 <=
            mid = left + (right - left) // 2  # 每轮重新算 mid
            if nums[mid] == target:
                return mid
            elif nums[mid] < target:
                left = mid + 1
            else:
                right = mid - 1
        return left   # 没找到，left 就是插入位置
```

### 关键改动对照

| 错误写法 | 问题 | 改成 |
|---|---|---|
| `mid=(right-left)/2`（循环外） | 浮点 + 不更新 | `mid = left + (right-left)//2`（循环内） |
| `else if` | 语法错 | `elif` |
| `while left<right` | 漏掉相等情况 | `while left<=right` |
| `left=mid` / `right=mid` | 死循环 | `left=mid+1` / `right=mid-1` |
| 无收尾 return | 找不到时没返回值 | `return left` |

---

## 三、为什么循环结束后 `left` 就是答案

### 循环不变量（面试推荐讲法）

在左闭右闭 `[left, right]` 写法里，始终维持：

- `left` 左边的所有元素（下标 `< left`）都 **< target**
- `right` 右边的所有元素（下标 `> right`）都 **≥ target**

逐次更新验证不变量成立：

- `nums[mid] < target` → mid 及其左边都比 target 小，故 `left = mid + 1`，保证「left 左边都 < target」。
- `nums[mid] >= target` → mid 及其右边都 ≥ target，故 `right = mid - 1`，保证「right 右边都 ≥ target」。

循环在 `left > right` 时结束（即 `left == right + 1`）。此刻：

- `left` 左边全部 < target
- `left`（即 `right + 1`）及右边全部 ≥ target

所以 `left` 正是**第一个 ≥ target 的位置**，恰好就是插入点的定义。

> 一句话总结：**循环结束时 `left` 是第一个不小于 target 的下标，这正是插入点。**

### 边界正确性（主动补一刀）

- target 比所有元素都大 → `left` 一路走到 `n`，返回数组长度 → 插在末尾。✅
- target 比所有元素都小 → `left` 始终为 0，返回 0 → 插在开头。✅

## 四、为什么 `left = mid` 会导致死循环

### 根源：整数除法向下取整

`mid = left + (right - left) // 2` 中 `//` 向下取整。当区间只剩两个元素（`right = left + 1`）时：

```
mid = left + (1 // 2) = left + 0 = left
```

即 `mid` 会**等于 left**，但永远不等于 right。

### 死循环过程（以 while left < right 为例）

```
left=2, right=3  →  mid = 2 (= left)
若 nums[mid] < target，执行 left = mid:
left = 2  →  left 还是 2，right 还是 3，区间完全没变 → 死循环
```

本质：**`mid` 等于 `left`，又把 `left` 更新成 `mid`，等于没更新。**

### 为什么 `+1` / `-1` 能解决

`mid` 已经判断过，要么是答案（已 return），要么可被排除。更新时把 mid 本身踢出区间，每轮至少缩小 1，必然收敛。

| 写法 | 当 `right=left+1`，`mid=left` 时 | 结果 |
|---|---|---|
| `left = mid` | left 赋成自己，区间不变 | 💀 死循环 |
| `left = mid + 1` | left 越过 mid，区间缩小 | ✅ 收敛 |

> 小贴士：`right = mid` 不一定死循环（因为 `mid` 永远不等于 `right`），真正危险的是 `left = mid`。但统一用 `left=mid+1` / `right=mid-1` 配 `while left<=right` 最稳，不用费脑子判断哪边会卡死。
