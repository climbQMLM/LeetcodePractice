# 1. Two Sum（两数之和）

## 题目核心思路

用哈希表存储「值 → 下标」的映射，边遍历边查找补数（`target - x`）。时间复杂度 O(n)，是本题的最优解法。

## 原始写法的问题

核心逻辑是对的，但漏了最关键的一步：**从来没有把元素存进 `idx` 字典里**。

因此 `idx` 永远是空的，`target - x in idx` 永远为 `False`，整个循环走完什么都没找到，函数最后返回 `None`。

## 修正后的代码

```python
class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        idx = {}
        for i, x in enumerate(nums):
            if target - x in idx:
                return [idx[target - x], i]
            idx[x] = i   # 关键：把当前值和下标存进去
```

## 细节说明

**1. 存字典的位置**

存字典这一步要放在 `return` 判断**之后**。先查再存，能保证不会把同一个元素用两次。

例如 `nums = [3, 3], target = 6`：如果先存后查，第一个 3 存进去后会立刻查到自己，导致错误。

**2. 返回顺序**

返回 `[idx[target - x], i]` 中：

- `idx[target - x]`：之前存入的下标（较小）
- `i`：当前下标（较大）

原写法 `[i, idx[target - x]]` 顺序反了。虽然 LeetCode 本题对顺序不严格，但习惯上「前小后大」更清晰。

## 复杂度

- 时间复杂度：O(n)
- 空间复杂度：O(n)
