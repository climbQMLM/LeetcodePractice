# 209. Minimum Size Subarray Sum

难度：Medium
链接：https://leetcode.com/problems/minimum-size-subarray-sum/

## 思维误区

收缩窗口时，先移动左指针，再用新的左指针去减窗口和——这样减掉的其实是"新的左端元素"，而不是"原本要移出窗口的那个元素"。

```python
# ❌ 错误写法：先移动指针，再用新位置的值去减
left += 1
s -= nums[left]   # 这里 nums[left] 已经是下一个元素，不是被移出窗口的元素
```

```python
# ✅ 正确写法：先用旧的左指针减掉对应元素，再移动指针
s -= nums[left]
left += 1
```

窗口是 `nums[left..right]`（闭区间），移出左端元素时必须先用**当前**的 `left` 取值做减法，再把 `left` 右移；顺序颠倒会导致减错元素，窗口和与实际窗口不再对应。

## 完整代码

```python
class Solution:
    def minSubArrayLen(self, target: int, nums: List[int]) -> int:
        left = 0
        ans = float('inf')
        s = 0
        for right, x in enumerate(nums):
            s += x
            while s >= target:
                if s - nums[left] >= target:
                    s -= nums[left]
                    left += 1
                else:
                    ans = min(ans, right - left + 1)
                    break
        return ans if ans != float('inf') else 0
```

时间复杂度 O(n)，空间复杂度 O(1)。
