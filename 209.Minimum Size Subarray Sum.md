# 209. Minimum Size Subarray Sum

难度：Medium
链接：https://leetcode.com/problems/minimum-size-subarray-sum/

## 思维误区（按踩坑顺序）

### 1. 不是真正的滑动窗口，而是每次暴力重新求和
```python
# ❌ 错误写法：每轮重新对切片求和，退化成 O(n^2)，且 left 从未移动
while right < len(nums):
    if sum(nums[left:right]) >= target:
        return right - left
    else:
        right += 1
```
```python
# ✅ 正确写法：维护一个增量变量 window_sum，扩张/收缩时只做加减
window_sum += nums[right]   # 扩张
window_sum -= nums[left]    # 收缩
```

### 2. 初始化时 left 和 right 指向同一个下标，导致重复计数
```python
# ❌ 错误写法：left==right==0 时，nums[0] 被加了两次
left, right = 0, 0
window_sum = nums[left] + nums[right]
```
```python
# ✅ 正确写法：窗口只有一个元素时，只统计一次
left, right = 0, 0
window_sum = nums[0]
```

### 3. 右指针扩张时数组越界
```python
# ❌ 错误写法：right 已经是最后一个下标时，仍然 right+=1 后访问 nums[right]
else:
    right += 1
    window_sum += nums[right]   # right 可能等于 len(nums)，越界
```
```python
# ✅ 正确写法：用 for right, x in enumerate(nums) 让右指针的扩张和取值天然同步，不会越界
for right, x in enumerate(nums):
    window_sum += x
```

### 4. 死循环：状态不变但循环条件仍然满足
```python
# ❌ 错误写法：right 到达末尾且仍不满足条件时，if/elif 都不执行，
# 任何变量都没变化，但 while right<len(nums) 仍为 True，永久卡住
elif right < len(nums) - 1:
    right += 1
    window_sum += nums[right]
```
```python
# ✅ 正确写法：用 for 循环遍历右指针，天然保证每轮 right 都会前进，不会卡死
for right, x in enumerate(nums):
    ...
```

### 5. ans 在不满足条件时也被无条件更新，污染结果
```python
# ❌ 错误写法：每轮循环顶部无条件更新 ans，不管 window_sum 是否达标
while right < len(nums):
    ans = min(ans, right - left + 1)   # 不满足 target 时也在更新
    if window_sum >= target:
        ...
```
```python
# ✅ 正确写法：只有在 s>=target 的分支内部才更新 ans
while s >= target:
    ans = min(ans, right - left + 1)
```

### 6. 收缩条件设计反了，导致内层 while 死循环或漏判最优解
```python
# ❌ 错误写法：s - nums[left] < target 时只更新 ans，没有 break，
# 状态不变但 while s>=target 仍为真，死循环
while s >= target:
    if s - nums[left] < target:
        ans = min(ans, right - left + 1)
    else:
        left += 1
        s -= nums[left]
```
```python
# ✅ 正确写法：用 else+break 明确退出收缩，避免死循环
while s >= target:
    if s - nums[left] >= target:
        s -= nums[left]
        left += 1
    else:
        ans = min(ans, right - left + 1)
        break
```

### 7. 收缩时先移动指针、再用新位置的值去减（最后修复的 bug）
```python
# ❌ 错误写法：先移动指针，再用新的左指针去减——减掉的其实是"新的左端元素"
left += 1
s -= nums[left]
```
```python
# ✅ 正确写法：先用旧的左指针减掉对应元素，再移动指针
s -= nums[left]
left += 1
```

## 完整代码

```python
class Solution:
    def minSubArrayLen(self, target: int, nums: List[int]) -> int:
        left = 0
        ans = float('inf')
        s = 0
        for right, x in enumerate(nums):
            s += x
            while s >= target:
                if s - nums[left] >= target:
                    s -= nums[left]
                    left += 1
                else:
                    ans = min(ans, right - left + 1)
                    break
        return ans if ans != float('inf') else 0
```

时间复杂度 O(n)，空间复杂度 O(1)。
